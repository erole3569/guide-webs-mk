# guide-webs-mk

这是一个用于归档和发布独立 HTML 页面的仓库。仓库内收录了多个独立网页文件，涵盖不同主题与用途，方便统一管理与访问。

## 项目简介

本仓库的主要目的是整理和存放各类独立的 HTML 页面。这些页面可能是工具页面、信息展示页、静态演示页等，彼此之间没有强关联，但统一存放在此处以便于维护和分享。

仓库本身不绑定任何特定域名或网站，所有页面均为独立 HTML 文件，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

```
guide-webs-mk/
├── pages/           # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/          # 页面共用的资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html       # 仓库首页（可选）
└── README.md        # 本文件
```

- `pages/`：每个独立页面单独存放于此，文件名应简洁明了。
- `assets/`：存放页面可能共享的样式、脚本和媒体资源。若页面有独立资源，也可在页面同级目录下创建 `assets` 子目录。
- `index.html`：可选的首页文件，用于导航或展示页面列表。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖后端服务。
- 页面之间相互独立，不共享状态或数据。
- 页面内容可能包含静态信息、交互功能或演示效果，具体以页面本身说明为准。
- 页面文件名建议使用英文小写字母、数字和连字符，避免特殊字符。

## 维护说明

- 添加新页面：在 `pages/` 目录下新建 HTML 文件，并视需要添加资源文件。
- 更新页面：直接编辑对应 HTML 文件，注意保持与其他页面的独立性。
- 删除页面：移除对应文件即可，无需额外操作。
- 如需更新共享资源（如 `assets/` 中的文件），请确保不会影响其他页面的正常显示。
- 建议每次更改后提交清晰明确的 commit 信息。

## 使用方式

你可以通过以下方式访问仓库中的页面：

1. **克隆仓库到本地**：`git clone <仓库地址>`，然后直接在浏览器中打开 HTML 文件。
2. **通过 GitHub Pages 访问**：启用仓库的 GitHub Pages 功能后，可通过 `https://<用户名>.github.io/guide-webs-mk/pages/<页面文件名>` 访问对应页面。

## 注意事项

- 本仓库仅用于存放 HTML 页面，不提供任何形式的担保或支持。
- 页面内容由贡献者自行负责，请确保不违反相关法律法规。
- 欢迎通过 Issue 或 Pull Request 提出改进建议或报告问题。
